# Session 1 (10:00 AM – 11:00 AM)

**File:** `Session_10_11.md`

## Objectives
- Understand the architecture of a web crawler.
- Identify the major components required for the crawler.
- Study how search engines discover webpages.

## Work Completed
- Researched the overall workflow of a web crawler.
- Identified the crawler pipeline:
  - Seed URLs
  - Frontier
  - Seen Store
  - Fetcher
  - Link Extractor
  - URL Normalizer
  - Page Storage
- Studied BFS and DFS.
- Selected BFS for better coverage and natural depth traversal.

## Design Decisions
- Modular architecture.
- Independent components.
- Testable interfaces.

## Outcome
Completed the high-level architecture and identified all major crawler modules.
